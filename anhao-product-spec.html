// 01

产品定位

不是约会App。不是Meetup。不是整点薯条。暗号是全新物种。

target
目标用户
  • 城市：杭州（冷启动）
  • 人群：22-35岁，互联网/科技从业者
  • 场景：工作日午餐、周末晚餐、想认识人但不想社恐
not_this
不是什么
  • 不是约会App（降低心理门槛）
  • 不是Meetup式大活动（保持亲密感）
  • 不是整点薯条（不固定6人桌）
core_diff
五大核心差异
暗号对接仪式 完全盲盒 需求驱动匹配 饭后保持神秘 心理题游戏化
  • 每人拿半句暗号（谜面+提示），面对面合力解谜才能「开箱」
  • 见面前不知对方是谁，只有渐进式线索
  • 用户定义"想吃什么、在哪吃、对方什么样"
  • 饭后不交换信息，想再见只能通过暗号再次匹配
  • 心理题既是画像工具，也是匹配入口
// 02

核心用户旅程

6步闭环。从注册到再次相遇。

01

注册

手机号 → 口味标签 → 3道心理题 → 进入首页。渐进实名：初期只需手机号，1v1需身份认证。

02

发出暗号

'日料，1公里内，要有趣。' — 填写约饭条件，结构化标签 + 自由文本「惊喜期待」栏，提交后进入双向匹配池。

03

匹配成功 → 渐进揭示

Push通知「有人接上了你的暗号！」→ 餐厅+时间。当天即时，每30分钟一条模糊线索。同时收到半句暗号。

04

赴约 — 暗号对接

A念谜面、B念提示 → 合力猜谜底 → 输入验证 →「暗号对上了！」→ 开箱揭示对方秘密 → 破冰 → AA结账。

05

饭后

评分 + 标签 +「想再约」。双方都点 → 缘分池（提高概率，不保证）。不交换任何信息。

06

日常 — 每日暗号题

一道心理题 →「有247人选了和你一样的答案」→ 答同题的人匹配优先。AI生成，渐进构建6维画像。

// 03

暗号对接系统

产品名就叫暗号。每次见面都在「对暗号」。品牌即体验。

user_a
你的暗号（谜面）
'什么东西越洗越脏？'
找到你的饭搭子，合力解开谜底。
user_b
你的暗号（提示）
'和水有关，但不是喝的'
找到你的饭搭子，合力解开谜底。

两人交流线索 → 猜出谜底 → 双方输入 → 「暗号对上了！」 → 开箱：揭示对方的一个秘密 → 自然破冰

riddle_db
谜语库
  • AI批量生成，人工审核入库
  • 难度：简单 / 中等 / 趣味
  • 3次错误给额外提示，不会卡住
multi_player
多人局
  • 2v2/4人局用统一主暗号词
  • 到场互报暗号确认身份
  • 不用谜语，保持简单
// 04

渐进揭示系统

当天即时匹配。每30分钟一条线索。制造期待，不暴露身份。

时间揭示内容来源
T+0餐厅名 + 时间 + 半句暗号匹配结果
T+30m「TA最近的回答：周末最想做的事是___」心理题
T+60m「TA的一个标签：咖啡上瘾」画像标签
T+90m「TA说了一句话：___」惊喜期待栏
到达暗号对接 → 开箱揭秘对方画像

// 06

匹配算法

三层漏斗。硬条件 → 标签 → 画像。服务端执行。

layer_1 // hard_filter
硬条件过滤
时间窗口重叠 距离范围交集 菜系偏好交集 桌型一致
layer_2 // tag_match
标签匹配
  • 双方结构化标签互满足度
  • 自由文本语义相似度（Phase 2）
layer_3 // soft_score
画像匹配
指标权重说明
心理画像相似度40%6维余弦相似度
信用分30%失约/举报/好评
历史评价20%饭后评分均值
缘分池+50%双方互标「想再约」

触发：每5分钟批处理 + 新用户即时匹配。缘分池：用户不知对方是否点了「想再约」，匹配分+50%但不保证。
