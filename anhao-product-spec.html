01

产品定位

不是约会App，不是Meetup式大活动，不是整点薯条。暗号是一个全新物种。

Target
目标用户
  • 城市：杭州（冷启动）
  • 人群：22-35岁，互联网/科技从业者
  • 场景：工作日午餐搭子、周末晚餐社交、想认识新朋友但不想社恐
Not This
不是什么
  • 不是约会App（降低心理门槛）
  • 不是Meetup式大活动（保持亲密感）
  • 不是整点薯条（不固定6人桌、不提前揭示身份）
Core Differentiators
五大核心差异
暗号对接仪式 完全盲盒 需求驱动匹配 饭后保持神秘 心理题社交游戏化
  • 每人拿半句暗号（谜面+提示），面对面合力解谜才能「开箱」
  • 见面前不知道对方是谁，只有渐进式线索
  • 用户定义"想吃什么、在哪吃、对方什么样"
  • 饭后不交换信息，想再见只能通过暗号再次匹配
  • 心理题既是画像工具，也是匹配入口
02

核心用户旅程

从注册到再次相遇，6步完整闭环。

1

注册

手机号注册 → 选择口味标签 → 回答3道趣味心理题 → 进入首页。渐进实名：初期只需手机号，1v1约饭需身份认证。

2

发出暗号

填写约饭条件——想吃什么、什么时候、几公里内、桌型（1v1/2v2/4人）、对对方的期待（结构化标签 + 自由文本「惊喜期待」栏）。提交后进入双向匹配池，系统后台持续匹配。

3

匹配成功 → 渐进揭示

Push通知「有人接上了你的暗号！」→ 展示餐厅+时间。当天即时匹配，每30分钟揭示一条模糊线索。同时收到自己的半句暗号（谜面或提示）。

4

赴约 — 暗号对接

A念谜面、B念提示 → 合力猜谜底 → 双方输入答案 → 系统验证 →「暗号对上了！」→ 开箱揭示对方的一个秘密 → 自然破冰开聊 → AA结账。

5

饭后

打分（1-5星）+ 反馈标签 +「想再约」按钮。双方都点则进入缘分池（提高再次匹配概率，不保证）。不交换任何信息，不揭示真实身份。

6

日常 — 每日暗号题

一道趣味心理题 → 回答后看到「有247人选了和你一样的答案」→ 答同题的人匹配时优先配对。题目由AI批量生成，渐进构建6维画像向量。

03

暗号对接系统

产品名就叫暗号，用户每次见面都在「对暗号」——品牌即体验。

User A
你的暗号（谜面）
「什么东西越洗越脏？」
找到你的饭搭子，合力解开谜底
User B
你的暗号（提示）
「和水有关，但不是喝的」
找到你的饭搭子，合力解开谜底

两人交流线索 → 猜出谜底「水」→ 双方输入 → 「暗号对上了！」 → 开箱：揭示对方的一个秘密（兴趣标签或心理题答案）→ 自然破冰

Riddle Library
谜语库
  • AI批量生成，人工审核入库
  • 难度分级：简单 / 中等 / 趣味
  • 分类：文字谜、脑筋急转弯、常识题、生活场景
  • 3次错误后给额外提示，确保不卡住
Multi-Person
多人局暗号
  • 2v2/4人局使用统一主暗号词
  • 到场互报暗号词确认身份
  • 不使用谜语（多人解谜效率低）
  • 保持简单直接
04

渐进揭示系统

当天即时匹配，每30分钟一条线索，制造期待但不暴露身份。

时间点揭示内容数据来源
匹配即时餐厅名 + 时间 + 自己的半句暗号匹配结果
+30min「TA最近的回答：周末最想做的事是___」心理题答案
+60min「TA的一个标签：咖啡上瘾」用户画像标签
+90min「TA说了一句话：___」惊喜期待栏
到达暗号对接 → 开箱揭秘对方画像

线索规则：从对方画像中随机抽取，不透露姓名/公司/精确年龄。新用户画像不足时，用通用趣味描述（如"TA今天心情不错"）。每次匹配线索不重复。

心理题与画像系统

让系统越来越懂用户，让答题本身成为社交入口。

Question Types
4类题目
类型示例
性格「朋友迟到30分钟，你会？」
价值观「周末最理想的度过方式？」
场景「火锅必点的第一道菜？」

6-Dim Vector
画像向量
  • 外向性 ←→ 内向性
  • 冒险性 ←→ 稳定性
  • 计划性 ←→ 随性
  • 独立性 ←→ 合群性
  • 感性 ←→ 理性
  • 传统 ←→ 新潮

余弦相似度匹配，"同频共振"策略。答同题的人优先配对。

06

匹配算法设计

三层漏斗，从硬条件到软画像，服务端执行。

Layer 1 — Hard Filter
硬条件过滤（必须全部满足）
时间窗口重叠 距离范围交集 菜系偏好交集 桌型一致
Layer 2 — Tag Match
标签匹配
  • 双方结构化标签互满足度（A对B的要求 + B对A的要求）
  • 自由文本语义相似度（Phase 2，AI理解）
Layer 3 — Soft Score
画像匹配（加权评分）
指标权重说明
心理题画像相似度40%6维向量余弦相似度
信用分30%失约/举报/好评积累
历史评价均分20%饭后评价积累
缘分池加成+50%双方互标「想再约」

触发机制：每5分钟批处理扫描 + 新用户即时匹配。匹配成功后从池中移除，超时自动过期。
缘分池：用户不知道对方是否点了「想再约」，下次同时在池中时匹配分 +50%，但不保证一定匹配上。

配置下发机制

登录时：GET /config → 完整配置 + 版本号 → 缓存本地

日常：客户端每次API请求携带 X-Config-Version: 4，服务端发现新版本 → 响应头通知 X-Config-Stale: true → 客户端静默拉取新配置

降级：本地缓存丢失/首次安装 → 客户端内置默认配置兜底

CI/CD 流水线

GitHub Actions 全链路自动化。PR测试 → 合并部署。

PR Pipeline
ci.yml
  • pnpm install (缓存)
  • ESLint + Prettier
  • TypeScript 类型检查
  • BDD 测试 (Cucumber)
  • 单元测试 (Vitest)
  • 组件测试 (Jest)

全部通过 → 允许合并

Deploy Pipeline
合并到 main
  • workers/ 变更 → wrangler deploy + D1 migrations + 烟雾测试
  • apps/mobile/ 变更 → expo export → OTA推送到自建EAS
  • packages/shared/ 变更 → 触发两者

分支策略：main 生产环境（保护分支）← feat/xxx 功能分支 / fix/xxx 修复分支，通过PR合并。
