## Expo Go 调试方案（下午）

### 问题：重复 EAS 构建太费配额
- EAS Build 付费额度 $20，已经用了 ~5 次构建
- 改用 Expo Go + Tunnel 模式做热重载开发

### Expo Go 原生模块兼容性问题
- **根本问题**：Expo Go 不支持自定义原生模块，只支持 Expo SDK 内置的
- 不兼容的模块（先后发现）：
  1. `expo-secure-store` → 替换为 `@react-native-async-storage/async-storage`（纯 JS）
  2. `@react-native-voice/voice` → 整个 `useSpeech.js` 改为 mock 实现
  3. `expo-crypto`（部分 API）→ `Crypto.digest` 替换为 `js-sha256`（纯 JS），`Crypto.getRandomBytes` 移除（tweetnacl 用内置 fallback）
- **关键教训**：Expo Go 模式下必须确保所有依赖都是纯 JS 或 Expo SDK 内置模块
- 替换后 App 应该只用纯 JS 依赖：tweetnacl, js-sha256, @react-native-async-storage/async-storage

### Expo 开发服务器
- 用 `EXPO_TOKEN` 环境变量登录避免交互提示
- Tunnel 模式需要 Expo 账号登录（匿名模式证书获取失败）
- Tunnel URL 格式：`exp://xxx-notjayson-8081.exp.direct`
- 老金不在家，不能用局域网模式（`exp://192.168.10.167:8081`）

### 安全注意
- 当前 PRNG 用 tweetnacl 内置 fallback（可能基于 Math.random），**不安全**
- 仅用于 Expo Go 调试阶段
- 正式构建必须恢复 `expo-crypto` + `expo-secure-store`
